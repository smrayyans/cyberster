3. Task 01 - Passive OSINT and Subdomain Enumeration

Cross-source enumeration is used to reduce blind spots and increase confidence in discovered assets.

subfinder -d target.com -silent -o subfinder.txt assetfinder --subs-only target.com > assetfinder.txt amass enum -passive -d target.com -o amass.txt cat subfinder.txt assetfinder.txt amass.txt | sort -u > master_subdomains.txt wc -l master_subdomains.txt sort subfinder.txt assetfinder.txt amass.txt | uniq -c | sort -nr | head
xlsx2csv dnsdumpster.xlsx > dnsdump.csv grep -oE "[a-zA-Z0-9.-]+\.target\.com" dnsdump.csv | sort -u > dnsdump.txt cat master_subdomains.txt dnsdump.txt crt_target.txt | sort -u > attack_surface.txt

Operational insight: repeated occurrence of the same asset across multiple tools indicates higher confidence and triage priority.

4. Task 02 - Infrastructure and Technology Profiling

Fingerprinting helps identify relevant attack paths and probable defensive controls.

whatweb https://sub.target.com > tech.txt whois target.com > whois.txt nslookup -type=A target.com nslookup -type=MX target.com nslookup -type=NS target.com nslookup -type=TXT target.com dig target.com ANY
site:target.com inurl:phpinfo.php site:target.com filetype:env site:target.com inurl:robots.txt site:target.com filetype:bak site:target.com filetype:zip site:target.com intitle:"index of"

Expected documentation: server/framework clues, DNS record mapping, CDN indicators, and any indexed sensitive resources.

5. Task 03 - GitHub Dorking and Secret Leak Checks

Public code and commit history are inspected for accidental credential exposure.

org:targetorg "API_KEY" org:targetorg "PASSWORD" org:targetorg "AWS_SECRET" "target.com" "API_KEY" filename:.env "target" filename:settings.py "SECRET_KEY"
git log -p git grep -i "password" $(git rev-list --all) git log -S "API_KEY" --all

Validation rule: differentiate real secrets from placeholders, and always inspect previous commits for removed but recoverable values.

6. Task 04 - Live Asset Filtering and Visual Recon

Focus recon on responsive targets to save time and surface high-value endpoints.

cat attack_surface.txt | httprobe --prefer-https > live-urls.txt # or cat attack_surface.txt | httpx -status-code -title -tech-detect -o live-httpx.txt cat live-urls.txt | aquatone -out ./aquatone_results # or gowitness file -f live-urls.txt --screenshot-path ./gowitness_screenshots

Report output should include: total assets found, total live hosts, notable panels, and obvious exposure indicators.
