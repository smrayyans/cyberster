What to Look For
You are NOT hunting for exploitation.
You are checking:
Is the value hardcoded?
Is it real-looking or test?
Is it commented out?
Is it part of documentation?
Example risky patterns:
const API_KEY = "sk_live_93ksk29..."
AWS_SECRET=AKIAIOSFODNN7...
password = "Admin@123" 

If a repository within is public:
Open the file.
Click History (top right).
Browse older commits.
“Even if a secret was removed later, Git preserves previous versions.”
Look for:
Added credentials in older commits
Secrets later removed
Config files changed
